# Документація

## Запустити додаток локально

1. `npm install`
2. `npm run dev`

## Загальна структура всіх ігор

Кожна гра побудована за схожим принципом і використовує наступні React концепції:

### Основні хуки (Hooks):

- `useState` - для управління станом компонента
- `useNavigate` - для навігації між сторінками
- `useEffect` - для виконання побічних ефектів (ініціалізація, таймери)
- `useMemo` - для мемоізації обчислень

### Типові стани в іграх:

1. **currentIndex/currentScene** - номер поточного питання/рівня
2. **feedback** - зворотний зв'язок ('correct', 'wrong', 'complete', null)
3. **score** - кількість правильних відповідей
4. **selected/clicked** - відстеження вибраних елементів

### Загальний потік гри:

1. Відображення питання/завдання
2. Вибір користувача
3. Перевірка правильності
4. Візуальний зворотний зв'язок
5. Затримка (setTimeout)
6. Перехід до наступного питання або завершення

## Загальні CSS класи

### Основні контейнери:

- `.game-container` - головний контейнер гри
- `.game-header` - заголовок з назвою та інструкціями
- `.choice-container` - контейнер для варіантів вибору

### Інтерактивні елементи:

- `.choice-card` - картка з варіантом відповіді
- `.choice-image` - зображення/емодзі у картці
- `.choice-label` - текст у картці

### Зворотний зв'язок:

- `.feedback.correct` - правильна відповідь (зелений)
- `.feedback.wrong` - неправильна відповідь (червоний)

### Спеціальні:

- `.beach-grid` - сітка для FindTheBall
- `.memory-grid` - сітка для FriendsMemory
- `.beach-card` / `.memory-card` - картки у відповідних іграх

### Кнопки:

- `.home-button` - кнопка повернення на головну
- `.next-button` - кнопка продовження/завершення

---

### Анімації:

Використовуються CSS анімації:

- `float` - плавне рух вгору-вниз
- `bounce` - підстрибування
- `celebrate` - святкова анімація

### Accessibility:

- Великі шрифти (1.8rem - 15rem)
- Контрастні кольори
- Великі зони кліку
- Чіткі візуальні індикатори

### Адаптивність:

- Використання `flexbox` та `grid`
- `flexWrap: 'wrap'` для мобільних
- Мінімальні ширини (`minWidth`)

## Підтримка та розширення

### Додавання нових предметів:

1. Додайте об'єкт до відповідного масиву
2. Переконайтеся, що всі властивості заповнені
3. Емодзі мають бути сумісні з усіма платформами

### Додавання нових ігор:

1. Скопіюйте структуру схожої гри
2. Змініть дані та логіку
3. Додайте маршрут у головному додатку
4. Створіть кнопку на головному меню

### Локалізація:

- Всі тексти винесені у константи
- Легко додати переклади
- Емодзі універсальні для всіх мов
